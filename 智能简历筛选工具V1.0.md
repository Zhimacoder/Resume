# 智能简历筛选工具 PRD 文档
**版本号**：V1.0  
**编写日期**：2026-04-20  


## 1. 项目概述
### 1.1 项目背景
企业招聘过程中，HR 需人工筛选大量简历，效率低且易受主观因素影响。为提升简历筛选效率，降低人工成本，需开发一款基于大模型的智能简历筛选工具，实现简历自动解析、匹配度分析及面试问题推荐。

### 1.2 产品目标
- 支持批量上传简历（≤10 份），自动完成 OCR 识别与信息脱敏。
- 基于大模型分析简历与 JD 的匹配度，输出结构化筛选结果。
- 提供 APIKey 配置页面，支持主流大模型灵活切换。

### 1.3 目标用户
企业 HR、招聘专员及需要快速筛选简历的业务人员。


## 2. 功能需求
### 2.1 功能模块总览
| 模块 | 功能描述 |
|------|----------|
| 主页（页面1） | JD 填写、简历上传、智能筛选触发 |
| 结果页（页面2） | 简历列表展示、详情查看、匹配度分析 |
| API 配置页（页面3） | 主流大模型 APIKey 配置、自定义模型配置 |


### 2.2 主页（页面1）
#### 2.2.1 功能说明
| 功能点 | 详细描述 |
|--------|----------|
| JD 填写 | 提供文本输入框，支持粘贴/手动输入岗位 JD，无字数限制。 |
| 简历上传 | - 支持格式：doc、docx、pdf；<br>- 数量限制：一次最多上传 10 份；<br>- 交互：支持拖拽上传或点击按钮选择文件，上传后显示文件名列表。 |
| 智能筛选按钮 | 点击后触发以下流程：<br>1. 对简历进行 OCR 识别（提取文本内容）；<br>2. 信息脱敏（隐藏手机号、邮箱、姓名等敏感信息）；<br>3. 调用大模型接口，传入 JD + 脱敏后简历文本；<br>4. 跳转至结果页（页面2）。 |

#### 2.2.2 页面布局
- 顶部：标题“智能简历筛选工具”；
- 中部：JD 输入框（占页面宽度 80%）；
- 下部：简历上传区域（拖拽区 + 文件列表）+ “智能筛选”按钮。


### 2.3 简历筛选结果页（页面2）
#### 2.3.1 功能说明
| 功能点 | 详细描述 |
|--------|----------|
| 左侧布局（30% 宽度） | - 展示简历名称列表，按匹配度从高到低排序；<br>- 点击简历名称，右侧切换至对应详情。 |
| 右侧布局（70% 宽度） | 展示当前选中简历的详细信息：<br>1. 简历名称；<br>2. 简历概述（大模型生成的简历核心信息摘要）；<br>3. 与岗位的匹配点（分点列出）；<br>4. 与岗位的不足点（分点列出）；<br>5. 最适合的 3 个面试问题（分点列出）。 |

#### 2.3.2 交互逻辑
- 页面加载时默认展示排名第一的简历详情；
- 左侧列表支持滚动，右侧详情支持滚动（内容过多时）。


### 2.4 APIKey 配置页（页面3）
#### 2.4.1 功能说明
| 功能点 | 详细描述 |
|--------|----------|
| 主流大模型配置 | 支持以下模型的 APIKey 输入：<br>- 百度文心、阿里千问、字节豆包、智谱 GLM、MinMax；<br>- 每个模型对应独立输入框，输入后保存至本地。 |
| 自定义模型配置 | 支持用户输入：<br>- 模型名称、API Endpoint、APIKey；<br>- 保存后可在筛选时选择自定义模型。 |
| 模型选择 | 提供下拉框，选择当前用于筛选的大模型（默认选中第一个配置的模型）。 |

#### 2.4.2 页面入口
主页右上角设置“API 配置”按钮，点击跳转至配置页。


## 3. 非功能需求
### 3.1 性能需求
- 单份简历 OCR + 大模型响应时间 ≤ 10 秒（网络正常情况下）；
- 批量上传 10 份简历时，总处理时间 ≤ 2 分钟。

### 3.2 兼容性需求
- 前端：支持 Chrome、Firefox、Edge 最新版本浏览器；
- 后端：Python 3.8+，兼容 Windows/Linux/macOS。

### 3.3 安全性需求
- APIKey 加密存储至本地配置文件，不明文传输；
- 简历数据仅在本地处理，不上传至第三方服务器（大模型调用除外）。

### 3.4 可用性需求
- 页面操作流程 ≤ 3 步（上传 → 筛选 → 查看结果）；
- 提供操作提示（如“上传中”“分析中”等加载状态）。


## 4. 页面流程设计
### 4.1 用户操作流程
1. 进入主页，填写 JD；
2. 上传简历（≤10 份）；
3. 点击“智能筛选”，等待处理；
4. 跳转至结果页，点击左侧列表查看简历详情；
5. （可选）点击“API 配置”，设置大模型 APIKey。


## 5. 技术架构
### 5.1 技术栈
| 层级 | 技术选型 |
|------|----------|
| 前端 | HTML + CSS + JavaScript（原生，无需框架） |
| 后端 | Python 3.8+，Flask/FastAPI（轻量级 Web 框架） |
| OCR 识别 | PyTesseract（本地 OCR）或 第三方 OCR API（如百度 OCR） |
| 文档解析 | PyPDF2（PDF）、python-docx（docx）、antiword（doc） |
| 大模型调用 | Requests（HTTP 请求调用大模型 API） |

## 7. 风险评估与应对
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| OCR 识别准确率低 | 简历解析失败 | 优先使用第三方 OCR API（如百度 OCR），本地 OCR 作为备选；支持用户手动修正识别结果。 |
| 大模型返回结果不稳定 | 匹配度分析质量差 | 设计 Prompt 模板，规范大模型输出格式；支持用户切换大模型。 |
| 简历数据隐私泄露 | 法律风险 | 本地处理简历，仅传输脱敏后文本至大模型；APIKey 加密存储。 |
