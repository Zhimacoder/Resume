# 智能简历筛选工具 PRD 文档
**版本号**：V1.1  
**修订日期**：2026-04-20  
**修订说明**：新增APIkey全局状态校验、未配置拦截提醒、页面数据保活机制，完善全流程交互闭环


## 1. 项目概述
### 1.1 项目背景
企业招聘过程中，HR 需人工筛选大量简历，效率低且易受主观因素影响。为提升简历筛选效率，降低人工成本，需开发一款基于大模型的智能简历筛选工具，实现简历自动解析、匹配度分析及面试问题推荐。

### 1.2 产品目标
- 支持批量上传简历（≤10 份），自动完成 OCR 识别与信息脱敏。
- 基于大模型分析简历与 JD 的匹配度，输出结构化筛选结果。
- 提供 APIKey 配置页面，支持主流大模型灵活切换。
- 新增**APIkey配置状态全流程管控**，未配置时强提醒+操作拦截，已配置时无干扰，同时保障用户操作数据不丢失。

### 1.3 目标用户
企业 HR、招聘专员及需要快速筛选简历的业务人员。


## 2. 功能需求
### 2.0 全局APIkey状态校验模块（前置核心模块）
#### 2.0.1 状态定义
| 状态 | 判定规则 |
|------|----------|
| 未配置状态 | 无任何大模型配置信息，或当前选中的默认模型无有效、非空的APIkey+Endpoint完整配置 |
| 已配置状态 | 当前选中的大模型已完成APIkey+Endpoint配置，且字段非空、格式校验通过 |

#### 2.0.2 全局校验规则
- 校验触发时机：页面初始化加载、用户触发核心操作、路由页面跳转、配置信息保存/修改后
- 校验优先级：全局状态校验优先于所有业务操作执行，未通过校验时拦截核心业务流程
- 状态同步：配置信息修改后，实时同步全局状态，所有页面无需手动刷新即可更新展示逻辑

| 模块 | 功能描述 |
|------|----------|

### 2.2 主页（页面1）
#### 2.2.1 基础功能说明
| 功能点 | 详细描述 |
|--------|----------|
| JD 填写 | 提供文本输入框，支持粘贴/手动输入岗位 JD，无字数限制。 |
| 简历上传 | - 支持格式：doc、docx、pdf；<br>- 数量限制：一次最多上传 10 份；<br>- 交互：支持拖拽上传或点击按钮选择文件，上传后显示文件名列表。 |
| 智能筛选按钮 | 点击后先执行APIkey状态前置校验，校验通过后触发以下流程：<br>1. 对简历进行 OCR 识别（提取文本内容）；<br>2. 信息脱敏（隐藏手机号、邮箱、姓名等敏感信息）；<br>3. 调用大模型接口，传入 JD + 脱敏后简历文本；<br>4. 跳转至结果页（页面2）。 |

#### 2.2.2 APIkey状态醒目提醒模块
- **展示位置**：页面顶部通栏，标题下方，固定展示，层级高于页面其他非弹窗元素
- **展示规则**：仅**未配置状态**下渲染展示；**已配置状态下完全隐藏，不占用页面空间、无任何占位**
- **视觉规范**：醒目警示样式（例：红色渐变背景、白色加粗字体、内边距12px、通栏100%宽度），不遮挡核心操作区
- **内容与交互**：
  - 文案固定：【⚠️ 请先完成APIkey配置，否则无法使用智能筛选功能】
  - 右侧附带主按钮：【去配置APIkey】，点击后直接跳转至API配置页（页面3）
- **更新规则**：用户从配置页返回主页时，实时校验配置状态，已完成配置则自动销毁该提醒横幅，无需用户手动刷新页面

#### 2.2.3 未配置状态操作拦截与弹框规则
- **拦截触发条件**：未配置APIkey状态下，用户点击【智能筛选】按钮时，优先触发拦截逻辑，**不执行任何OCR、文件解析、接口调用等后续业务流程**
- **弹框规范**：
  - 弹框层级：页面最高层级，居中展示，遮罩覆盖全页面
  - 标题：缺少APIkey配置
  - 正文：当前未完成大模型APIkey有效配置，无法执行智能筛选，请先完成配置后再操作。
  - 按钮组：
    1. 【取消】：关闭弹框，停留在当前页面，**完整保留用户已填写的JD内容、已上传的简历文件，不做任何清空/重置**
    2. 【去配置APIkey】：主按钮，点击后直接跳转至API配置页（页面3）
- **补充规则**：未配置状态下，不限制用户填写JD、上传简历的操作，仅拦截【智能筛选】核心提交动作

#### 2.2.4 页面数据保活机制（核心约束）
- **保活数据范围**：用户在页面1已填写的JD文本内容、已上传的简历文件列表、文件解析临时缓存
- **保活生效场景**：
  1. 从页面1跳转至API配置页（页面3）时，自动触发数据缓存
  2. 从API配置页（页面3）返回页面1时，自动触发数据回填，页面不执行全局刷新
- **技术实现规则**：
  1. 采用浏览器`sessionStorage`作为核心缓存载体，JD文本直接存储，简历文件信息通过`FileReader`缓存基础信息与临时读取结果，确保跳转后可完整恢复
  2. 缓存有效期：当前浏览器会话全程有效（不考虑关闭浏览器、清除缓存等极端情况）
  3. 仅当用户主动点击【清空内容】、完成智能筛选全流程、手动刷新页面时，才可清空缓存；页面跳转、配置往返操作绝对不触发缓存清空
- **回填规则**：返回页面1时，先读取缓存数据，完整回填至JD输入框、恢复已上传的简历文件列表与展示，再执行APIkey状态校验，全程无数据丢失

#### 2.2.5 页面布局
- 顶部：标题“智能简历筛选工具” + 右上角【API配置】固定入口按钮
- 警示通栏：APIkey未配置时展示，已配置时不渲染
- 中部：JD 输入框（占页面宽度 80%）
- 下部：简历上传区域（拖拽区 + 文件列表）+ “智能筛选”按钮

## 3. 非功能需求
### 3.1 性能需求
- 单份简历 OCR + 大模型响应时间 ≤ 10 秒（网络正常情况下）；
- 批量上传 10 份简历时，总处理时间 ≤ 2 分钟；
- 页面跳转与数据回填耗时 ≤ 300ms，无感知刷新。

### 3.2 兼容性需求
- 前端：支持 Chrome、Firefox、Edge 最新版本浏览器；
- 后端：Python 3.8+，兼容 Windows/Linux/macOS。

### 3.3 安全性需求
- APIKey 加密存储至本地配置文件，不明文传输；
- 简历数据仅在本地处理，仅传输脱敏后文本至大模型；
- 本地缓存数据仅在当前会话有效，关闭浏览器后自动清除。

### 3.4 可用性需求
- 页面操作流程 ≤ 3 步（上传 → 筛选 → 查看结果）；
- 提供操作提示（如“上传中”“分析中”等加载状态）；
- 配置往返操作无数据丢失，用户操作零成本回溯。


## 4. 页面流程设计
### 4.1 完整用户操作流程
1. 进入主页，系统自动执行APIkey状态校验
   - 未配置：展示顶部警示通栏，不限制填写/上传操作
   - 已配置：无提醒，正常使用所有功能
2. 用户填写岗位JD、上传简历文件
3. 用户点击【智能筛选】按钮
   - 未配置：弹出拦截弹框，用户可选择取消停留，或去配置页完成配置
   - 已配置：正常执行简历解析、OCR、脱敏、大模型调用流程
4. 跳转至结果页，点击左侧列表查看简历详情
5. （配置流程）用户点击配置入口跳转至API配置页，完成配置并保存，点击返回主页，已填写/上传的数据完整保留，警示提醒自动消失


## 5. 技术架构
### 5.1 技术栈
| 层级 | 技术选型 |
|------|----------|
| 前端 | HTML + CSS + JavaScript（原生），sessionStorage/localStorage做数据缓存与状态管理 |
| 后端 | Python 3.8+，FastAPI（轻量级Web框架） |
| OCR 识别 | PyTesseract（本地OCR）/ 百度OCR API |
| 文档解析 | PyPDF2（PDF）、python-docx（docx）、antiword（doc） |
| 大模型调用 | Requests（HTTP请求调用大模型API） |
| 配置存储 | 后端本地加密JSON配置文件，前端同步配置状态缓存 |

### 5.2 前端核心逻辑约束
- 全局状态管理：通过原生JS维护单例的配置状态对象，所有页面共享同一份状态数据
- 数据保活：页面跳转前自动执行缓存写入，页面加载时优先读取缓存回填，再执行页面渲染
- 状态同步：配置保存后，通过`postMessage`或本地存储监听事件，实时同步主页状态，无需刷新页面

## 7. 风险评估与应对
| 风险 | 影响 | 应对措施 |
|------|------|----------|
| OCR 识别准确率低 | 简历解析失败 | 优先使用第三方 OCR API，本地 OCR 作为备选；支持用户手动修正识别结果。 |
| 大模型返回结果不稳定 | 匹配度分析质量差 | 设计固定Prompt模板，规范大模型输出格式；支持用户一键切换大模型。 |
| 简历数据隐私泄露 | 法律风险 | 本地处理简历，仅传输脱敏后文本至大模型；APIKey 加密存储，不明文传输。 |
| 页面跳转数据丢失 | 用户体验受损 | 双重缓存机制：全局变量+sessionStorage，跳转前强制写入缓存，返回时优先读取回填，覆盖所有往返场景。 |
| 配置状态不同步 | 逻辑异常 | 配置保存后触发全局状态广播，所有页面实时监听状态变更，自动更新展示与交互逻辑。 |
